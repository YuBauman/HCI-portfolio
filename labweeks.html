Temperature change interaction
I personally started exploring the temperature interaction and worked with Arduino kit to make different LED lights light up depending on the temperature. As the input was shown in random number value and not celcius, it was difficult to makes sense out of it to work with it further. I needed to convert this value into temperature value, so it was quite challenging to figure out how to convert input from a thermistor voltage divider to a temperature value - celsius. By the end of the day, I managed to build a code that works with celsius temperature, using the formula that represents linear relation between temperature value and output voltage, as thermistos resistance decreased when the temperature increases.

              void getTemp(float * t)
              {
              
                  // Converts input from a thermistor voltage divider to a temperature value.
                  // The voltage divider consists of thermistor Rt and series resistor R0.
                  // The value of R0 is equal to the thermistor resistance at T0.
                  // You must set the following constants:
                  //                  adcMax  ( ADC full range value )
                  //                  analogPin (Arduino analog input pin)
                  //                  invBeta  (inverse of the thermistor Beta value supplied by manufacturer).
                  // Use Arduino's default reference voltage (5V or 3.3V) with this module.
                  //
              
                const int analogPin = 0; // replace 0 with analog pin
                const float invBeta = 1.00 / 3380.00;   // replace "Beta" with beta of thermistor
              
                const  float adcMax = 1023.00;
                const float invT0 = 1.00 / 298.15;   // room temp in Kelvin
              
                int adcVal, i, numSamples = 5;
                float  K, C, F;
              
                adcVal = 0;
                for (i = 0; i < numSamples; i++)
                 {
                   adcVal = adcVal + analogRead(analogPin);
                   delay(100);
                 }
                adcVal = adcVal/5;
                K = 1.00 / (invT0 + invBeta*(log ( adcMax / (float) adcVal - 1.00)));
                C = K - 273.15;                      // convert to Celsius
                t[0] = K; t[1] = C;
                return;
              }

                Serial.begin(9600);
              }

Button interaction - light up LED strip

Day 6

Solving batteries issue
Today we started the day with rearrangement of the batteries to fix the issue with power supply for LED strips. We added power supply in batteries in the beginning of each LED strip to make sure that each of them has enough power to operate properly. It was a bit challenging, as we had to connect properly all the wires and make sure that everything is connected and grounded. After all, we managed to make it work, so for LED strips to work properly we used a battery holder that included four AA batteries that calculated to 6V.

Youtube shorts: Video of working LED strips with the batteries

Nevertheless, we encountered another problem. We found out that our LED strips consume 3.6A that made them very bright, but it meant that our batteries would die in an hour of lights being constantly on. That would mean that we have to change the batteries every hour, and considering that we used 12 batteries at the same time to power all LED strips in the circuit, even just for expo we would have to use around 50 batteries. That was too expensive and unsustainable for us, so we started looking for another solutions. With the help of Leiden students, we figured out that we could use the power supply that can be plugged in an outlet and provide around 10A, that would be enough for the whole LED strip we used.

Changing capacitive touch sensitivity
This day we figured out, that our capacitive touch doesn’t work properly with everyone. Sometimes the sensor was not sensitive enough that caused delay in reaction, or didn’t react on the touch at all. For that reason, I was working on changing the code in a way, that the sensitivity of capacitive touchboard increased. I managed to work it out and change the code in a way that people who had problems with getting reaction on capacitive touch, could interact with our touchpoints without the problems. We also found out that sometimes specific shoes might prevent capacitive touch to work properly, therefore it was a good idea for us to create the floor with pillows, so people lift their feet from the ground and then capacitive touch doesn’t depend on that.

Product

Escape Hut

Description: Our product allows people to escape the outside world and create their own universe according to their current mood. We all built blanket houses in our childhoods to be able to spend time there on our own or with our loved ones. We always felt safe and happy in this environment, and we wanted to bring back these memories to our visitors. We created the escape hut that brings you to another universe with colorful cloudy sky, music and abstract visuals. The interaction is open-ended, so users can explore the possibilities of different environments as long as they want. There are 6 touch points to change the colors of the sky, 4 touch points to control the sounds/music, and 1 touch point for the visuals that are part of the sky environment. Many possible combinations of lights, music and visuals, that trigger users’ interest and playfulness to touch around and explore the beatiful outcomes.



How it works: The cloudy sky is divided in 6 parts with different LED colors. Each touch point for lights has two color modes, so by touching these touch points users can witch between two colors and this way create the combination of colors on the sky that they like the most. There are possiblities for more warm colors like yellow and orange, or more cold colors like purple and blue.

The installation also has four touchpoints for the sounds, that were divided in beats, guitar sounds, and two touch points for melodies. Each touch point contains 3 different variations of sounds, so the users can switch between the sounds in one touch point and combine different sound by activating several touch points! They can also turn off completely some of the sounds if they want.

Last touch point is activating the abstract visuals in the middle of the sky, and has several abstract animated illustrations being drawn in processing. The users can switch between different visuals.

The sensors of each touch point are connected to the MPR121 capacitive touch board, so the interaction works on the capacitive touch. We used two Arduino boards, one was connected to the LED strips, and another board is connected to the sounds sensors and visuals sensor.
